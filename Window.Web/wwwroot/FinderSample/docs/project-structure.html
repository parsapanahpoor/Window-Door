پوشه / نام فایل توضیحـات
Finder این پوشه شامل تمام فایل های مربوط به قالب است.
Finder / src این پوشه همه فایل‌های قالب را مانند .pug، .scss، .js را نگه می‌دارد که سپس در پوشه dist پردازش می‌شوند (کامپایل یا کوچک می‌شوند).
Finder / dist پوشه حاوی تمام فایل‌های قالب مانند html، .css، .js است که از پوشه src با استفاده از Node.js و Gulp پردازش (کامپایل/مینیفی‌شده) شده‌اند.
Finder / dist / vendor این پوشه حاوی همه فایل‌های .js / .css است.
Finder / package.json فایل حاوی meta data در مورد برنامه یا ماژول شما است. مهمتر از همه، فهرستی از وابستگی‌ها را برای نصب از npm هنگام اجرای نصب npm دارد.
Finder / gulpfile.js فایل پیکربندی برای کتابخانه Gulp است. این شامل تمام وظایفی (taskهایی) است که می خواهید با Gulp انجام دهید. در مورد آن از بخش Working with Gulp یا در مستندات رسمی Gulp بیشتر بیاموزید.
Finder / src / js این پوشه حاوی فایل theme.js و کامپوننت ها با همه ماژول ها است. theme.js به دو فایل dist/js/theme.js و dist/js/theme.min.js پردازش می‌شود (کامپایل / کوچک‌سازی شده).
Finder / src / scss این پوشه حاوی تمام فایل های sass است و نتیجه کامپایل آن در پوشه dist/css/ قرار خواهد گرفت.
Finder / src / templates این پوشه حاوی تمام فایل‌های pug است که در پوشه های dist, dist/components و dist/docs به HTML کامپایل شده اند.
نکته: اگر از تنظیمات Pro Front-End با Node.js و Gulp استفاده نمی کنید و می خواهید مستقیماً فایل های html / css / js را ویرایش کنید، باید به دنبال پوشه dist باشید.

چه با Node.js و Gulp کار کنید یا نه، همچنان می‌توانید از پیش پردازنده‌های HTML و CSS بهره ببرید. برای یادگیری میتوانید بر روی این لینک کلیک کنید : استفاده از Pug و Sass
